Proactive HMI System


An innovative car HMI system contains multi-sensor interior hardware and advanced infotainment software built for customer progressive feature test. Aimed to practice and evaluate our latest ideas of automobiles in the near future.

The project mainly includes the following phases:

Prophase Research

Customer survey, competitor review and cabin user interview.

HMI Design

Prophase Report analyse, information architecture, hardware and software interface design.

Test Platform Build

Specs documentations, software and hardware development, sandbox development.

User Simulator Test

Scenario-based user driving simulation, data collection and user interview, recursive report.

Proactive Interactions

Driving safety is one of the kernel aspects I concerned about in the beginning. Modern HMI system usually provides complicated functions and causes extra distractions, and the screen-based interface is also complained for attention disturbing. Despite involving autonomous ADAS, I’m trying to design a new mechanism that enables the system present or launch the proper functions and data without user manipulation. That is a better system with machine-initiative proactive interactions.

proactive system mechanism

An ideal process of proactive behavior contains three phases. First, the process has to be triggered by a kind of input or data changes. For example, the interior face detector detected a new passenger. Also, the trigger can be a group of various data such as current time and the memory of location at this time in other days. Next, the system is able to build a perception model of user behavior, the environment, and the context. For example, the driver is talking with someone in the cabin and the music volume is too high. Finally, the system would behave properly according to the perception model such as turning down the music in the former example.

proactive interaction framework

Users access the HMI system in a new way that requires less manual input. Based on the mechanism, I developed a new proactive interaction framework to design features and interfaces for the proactive HMI. The user input cell has been changed to the processing and perception cell, so the manual input has been one the perception models. Another main change is the confirmation mechanism of the interaction process. It indicates the whether the process will be confirmed manually or will execute autonomously. The following table shows some examples of the functional varieties based on the framework.

proactive interactions


Smart Ambient

Smart Ambient adjust the cabin ambient and atmosphere automatically via the lighting system, the HVAC system, and the fragrance system. Cooperated with the face detection system, the skin temperature sensor, and the environmental multi-sensor system, the HMI system is able to turn down the music, adjust the ambient light and fragrance, and change the zone temperature automatically when it’s necessary.

The system also presents feature predictions according to the circumstances. It is like a more active virtual assistant who is able to start a conversation or an interaction process instead of waiting for command only. For example, the navigation system will recommend the most possible destination for now and suggest the POI search. This kind of feature is designed in almost every part of the infotainment system to reduce user manipulations.

Clear-view Instrument Cluster

Designing from ground up, I noticed that a large digital instrument cluster fixed behind the steering may not be a good solution. So I designed a slim, flexible instrument cluster that covers nothing outside the windshield, and with all the needed cruising information. The display is clearer than the HUD and enables the same perfect view. Plus, a classic mode is designed to show a more traditional and complex interface.

Clear-view instrument cluster


Classic Mode

For Auto-pilot mode and old school drivers, the instrument cluster will slide up to an 10-inch display. And the Classic Mode instrument cluster is able to show a larger navigation map and a complete speedometer.

The content of the instrument cluster also changes according to the driving mode. For instance, a tachometer will be shown in the sport gearbox mode.

Assistance Display

An assistance display on the dashboard is designed to display the navigation information and some other necessary notifications. By expanding the display area, the interface logic is simplified and thus reduces unnecessary menu navigation.

Assistance Display


Extended Map

With the extended map view, drivers not longer need to go back and forth in the navigation application. The information display and data input are separated from a single-thread interface and makes the interface flat and clear. Also, the copilot will feel easier to search POI without disturbing the current navigation.

Driving Simulation Test Platform

In order to have a more immersive test system and get better user feedback, the engineering team and I designed a real-car-based driving simulation test platform. The platform features a curve projection screen with realtime 3D sandbox driving system and a real cabin with advanced customizable HMI system. The proactive HMI system will be developed and deployed on the platform for further test and modification.
