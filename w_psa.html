psa-tongji hmi design research


Next generation human-machine interface research sponsored by Peugot Société Anonyme. Aimed to find a better way to help drivers and passengers communicating with PSA automobiles in the future. I was enrolled in several phases of the project including design study, information architecture, UI and visual design, guidelines and documentations, and cockpit prototype development.

The crew designed a Card-selection test to get user preference data. The data and the interview reports became part of the foundation of the following works of my module.

Cooperated with the market service agency, we've conducted a feature-based competitor HMI hardware and software review. Models targeted to the similar customer groups were selected and fully analysed via on-site experience.

We also developed a HMI feature evaluation system in order to figure out the key features for the target groups according to the massive survey data from competitor review and prophase user tests.

HMI User Experience Grid


The position on the horizontal axis shows a relative scale from functional, practical feeling or features to emotional, interesting feeling or features. The vertical axis shows a relative scale from energetic, movable feeling or features to still, durable feeling or features. The Z-axis indicates the user expectation of interface performance.

Interface Performance Expectation


The 6 aspects indicate the key factors of interface performance. Including the speed of feedback animations, the speed of data computing or loading, the precision of the UI elements, the amount of manipulation steps, the consistency between the interaction of the feature and other systems in the cabin, and the quality of the sensory feelings of the interactions.

Here is an example of using the system to figure out key features for a specific user target group from the result of design study.

Due to the design study result, we started designing the system interfaces with two different target group orientations. The two system differs with each other from ground up, even the information architectures.

In order to get more precise feedback from all our test group users to make a better design recursion, we've developed hi-fi HMI prototypes based on the PSA P2S framework with the help from the computer science students.

prototype for target group test a


Prototype A has an universal interaction framework that supports both button-knob and touch screen inputs. The textured UI elements with physical metaphors maintain consistent cabin experience.

prototype for target group test b


The flexible interfaces make Prototype B more customizable and powerful for the youngster target group users. The system contains a set of modern visual language design and a global sharing feature across all the system apps with other IoV features integrated.
